프론트엔드 / HTML — 기초부터 탄탄하게

HTML은 웹 문서의 구조(Semantics)를 정의하는 마크업 언어야. CSS가 “모양”, JS가 “행동”이라면, HTML은 “뼈대와 의미”를 만든다.

1) 최소 골격

<!doctype html>
<html lang="ko">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>문서 제목</title>
  </head>
  <body>
    콘텐츠...
  </body>
</html>

2) 시맨틱 태그 핵심

  • <header> 페이지/섹션의 머리말
  • <nav> 내비게이션 메뉴
  • <main> 문서의 고유 핵심 콘텐츠(페이지당 1개)
  • <section> 주제별 구획, 보통 제목 동반
  • <article> 독립 배포 가능한 글(게시글/뉴스 등)
  • <aside> 보조/사이드 정보
  • <footer> 꼬리말(저작권/연락처 등)

3) 문서 예시(시맨틱 + 접근성)

<main id="content">
  <article aria-labelledby="post-title">
    <h1 id="post-title">HTML 입문</h1>
    <p>HTML은 웹의 뼈대입니다.</p>
  </article>
</main>

4) 텍스트 & 링크 & 미디어

  • 텍스트: <h1~h6>, <p>, <strong>, <em>
  • 링크: <a href="/path" rel="noopener">…</a>
  • 이미지: <img src="…" alt="대체텍스트"> (의미 있는 이미지는 alt 필수)
  • 비디오: <video controls><source src="movie.mp4"></video>

5) 폼(기본)

<form action="/signup" method="post">
  <label for="email">이메일</label>
  <input id="email" name="email" type="email" required />
  <button type="submit">제출</button>
</form>

6) 표(테이블) — 캡션과 헤더

<table>
  <caption>월간 방문자 수</caption>
  <thead><tr><th scope="col">월</th><th scope="col">방문자</th></tr></thead>
  <tbody><tr><th scope="row">1월</th><td>12,400</td></tr></tbody>
</table>

7) 접근성 체크리스트(아주 중요)

  • 이미지에 alt 제공(장식용은 alt="")
  • 헤딩 레벨 순서 유지(h1 → h2 → h3…)
  • 양식 컨트롤은 <label for>로 연결
  • 색만으로 정보 전달하지 않기(텍스트/아이콘 병행)
  • 대체 키보드 포커스 가능(기본 HTML 컨트롤 활용)

8) SEO 기본

  • <title>, <meta name="description"> 적절히 작성
