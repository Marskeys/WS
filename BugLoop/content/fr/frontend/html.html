HTML ACCUEIL

HTML est un langage de balisage qui définit la structure et la signification des pages web. CSS/JS se superposent à HTML.

<!DOCTYPE html>
      <html lang="fr">
      <meta charset="utf-8">
      <title>Ma première page</title>
      <h1>Bonjour</h1>
      <p>HTML est le plan du web.</p>

DOMBasesBoilerplate

Introduction

Le navigateur analyse le HTML pour créer un arbre DOM, puis applique le CSS pour dessiner un arbre de rendu.

  • Une structure sémantique est avantageuse pour le SEO, l'accessibilité et la maintenance.
  • La norme est mise à jour via le Living Standard (WHATWG).
  • Vérifiez la compatibilité sur MDN ou Can I use.

Bases

  • <!DOCTYPE html>: Déclaration HTML5.
  • lang, charset, viewport sont essentiels.

Éléments

Élément = balise d'ouverture + contenu + balise de fermeture. Les éléments vides n'ont pas de balise de fermeture (ex : <img>).

<p>Un paragraphe</p>
      <img src="/path/cat.jpg" alt="Un chat">

Attributs

<a href="/docs" target="_blank" rel="noopener">Documentation</a>
      <input type="checkbox" checked> Les attributs booléens existent sans valeur = vrai

Titres

<h1>Titre principal</h1>
      <h2>Sous-titre</h2>
      <h3>Sous-sous-titre</h3>

Paragraphes

<p>Premier paragraphe</p>
      <p>Deuxième paragraphe</p>
      <hr>
      <p>Troisième<br>avec un saut de ligne</p>

Styles

<p style="color:tomato; font-weight:bold">Avertissement</p>

Les styles en ligne sont rapides mais difficiles à maintenir → CSS est recommandé.

Liens

<a href="https://example.com" target="_blank" rel="noopener">Lien externe</a>
      <a href="#blt-tables">Sauter à une section sur cette page</a>

Images

<img src="/img/photo.jpg" alt="Un chien heureux" width="420">

L'attribut alt est essentiel pour l'accessibilité et le SEO.

Listes

<ul><li>Liste à puces</li><li>deuxième puce</li></ul>
      <ol><li>Liste numérotée</li><li>deuxième élément</li></ol>
      <dl><dt>Terme</dt><dd>Définition</dd></dl>

Tableaux

<table>
        <caption>Ventes mensuelles</caption>
        <thead><tr><th>Mois</th><th>Ventes</th></tr></thead>
        <tbody>
          <tr><td>Jan</td><td>100</td></tr>

        </tbody>
      </table>

Formulaires

<form action="/submit" method="post">
        <label for="email">E-mail</label>
        <input id="email" name="email" type="email" required>
        <button type="submit">Soumettre</button>
      </form>

La liaison label[for] ↔ input[id] est essentielle pour l'accessibilité.

Sémantique

<header>En-tête</header>
      <nav>Navigation</nav>
      <main><article>Article</article></main>
      <aside>Barre latérale</aside>
      <footer>Pied de page</footer>

Block vs Inline

  • Block: Occupe toute la largeur disponible (ex. : div, p, h1).
  • Inline: N'occupe que l'espace nécessaire (ex. : span, a, img).

id vs class

<div id="hero">Identifiant unique, un par page</div>
      <div class="tag">Commun à plusieurs éléments</div>

<head> & Meta

<meta name="description" content="Résumé">
      <meta property="og:title" content="Titre pour les réseaux sociaux">
      <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

Chemins de fichiers

./img/cat.jpg    (Dossier actuel)
      ../assets/a.js  (Dossier parent)
      https://cdn.example.com/lib.css (Absolu)

Responsive

<meta name="viewport" content="width=device-width, initial-scale=1">

Le panneau n'a besoin que de changer sa hauteur, donc la largeur suit la mise en page existante.
