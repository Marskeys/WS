HTML 완전 정복: 웹 페이지의 뼈대를 세우는 기초

HTML (HyperText Markup Language)은 웹 페이지의 뼈대이자 구조를 정의하는 마크업 언어이다. HTML은 시각적인 디자인이나 동적인 동작을 담당하는 언어가 아니며, 오직 웹 브라우저에게 "이 콘텐츠가 무엇이며, 어떻게 배치되어야 하는지"에 대한 설계도 역할을 수행한다.

1. 웹 개발 3요소와 HTML의 지위

웹은 HTML, CSS, JavaScript라는 세 가지 핵심 언어로 구성된다. 이 세 가지 언어는 각각 독립적인 역할을 수행하며, 상호 보완적으로 작용하여 하나의 완전한 웹 페이지를 구성한다.

  • HTML (HyperText Markup Language): 웹 페이지의 구조와 콘텐츠를 담당한다. 모든 웹 페이지는 HTML을 기반으로 작성된다.
  • CSS (Cascading Style Sheets): HTML로 만들어진 구조에 스타일과 디자인을 입히는 역할을 한다. 색상, 글꼴, 레이아웃 등 시각적 요소를 정의한다.
  • JavaScript: 웹 페이지에 동적인 기능과 상호작용을 추가한다. 사용자의 클릭에 반응하거나 콘텐츠를 동적으로 변경하는 등의 기능을 담당한다.

2. HTML 문서의 기본 골격과 필수 요소

모든 HTML 문서는 웹 표준에 따라 정해진 기본 구조를 갖추고 있다. 이 구조는 브라우저가 문서를 올바르게 해석하고 렌더링하는 데 필수적이다. 

<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>문서 제목</title>
  </head>
  <body>
    <!-- 실제 화면에 보이는 내용 -->
    <h1>안녕하세요, HTML!</h1>
    <p>이것은 HTML 문서의 예시입니다.</p>
  </body>
</html>
  • <!DOCTYPE html>: 문서 유형을 선언하는 태그로, 이 문서가 HTML5 표준을 따르고 있음을 브라우저에 알린다.
  • <html>: HTML 문서 전체를 감싸는 루트(Root) 요소이다. lang 속성은 문서의 언어를 명시하여 접근성과 SEO에 도움을 준다.
  • <head>: 문서의 메타데이터를 담는 영역이다. 브라우저 화면에 직접 표시되지 않으며, 문서에 대한 정보(제목, 문자 인코딩, 외부 파일 연결 등)를 포함한다.
  • <body>: 웹 브라우저 화면에 실제 표시되는 모든 콘텐츠(텍스트, 이미지, 링크 등)를 포함하는 영역이다.
  • <meta charset="UTF-8">: 문서의 문자 인코딩을 지정한다. 한글을 포함한 다양한 문자를 올바르게 표시하기 위해 필수적이다.
  • <title>: 브라우저 탭에 표시되는 문서의 제목을 정의한다.

3. 브라우저 렌더링 과정: HTML이 웹 페이지가 되기까지

HTML 파일은 다음의 복잡한 과정을 거쳐 우리가 보는 웹 페이지로 변환된다.

  1. 리소스 요청 및 수신: 사용자가 URL을 입력하면, 브라우저는 웹 서버에 HTML 파일을 요청하고 응답을 받는다.
  2. 파싱 (Parsing): 브라우저의 HTML 파서가 HTML 코드를 위에서부터 순서대로 읽어 내려간다.
  3. DOM 트리 생성: 파싱 과정을 통해 브라우저는 HTML 태그들을 계층적 구조의 객체 모델인 DOM (Document Object Model) 트리로 변환한다.
  4. CSSOM 트리 생성: HTML에 연결된 CSS 파일을 파싱하여 CSSOM (CSS Object Model) 트리를 생성한다.
  5. 렌더 트리 생성: DOM 트리와 CSSOM 트리를 결합하여 화면에 표시될 요소와 그 스타일 정보를 모두 담은 렌더 트리 (Render Tree)를 만든다.
  6. 레이아웃 (Layout): 렌더 트리를 기반으로 각 요소의 정확한 위치와 크기를 계산한다.
  7. 페인팅 (Painting): 계산된 위치와 크기 정보를 바탕으로 실제 화면에 픽셀을 그리는 작업이다.
  8. JavaScript 실행: JavaScript가 DOM을 조작하여 웹 페이지의 내용을 동적으로 변경하거나 기능을 추가한다.

4. 주요 HTML 태그 목록 및 분류

HTML5에서는 문서의 목적에 맞게 다양한 태그를 제공한다. 다음은 웹 개발에서 자주 사용되는 주요 태그들을 기능별로 분류한 목록이다.

4.1. 문서 구조 및 시맨틱 태그

  • <header>: 페이지의 머리말이나 탐색 메뉴를 포함한다.
  • <nav>: 웹 페이지의 주요 탐색 링크를 포함한다.
  • <main>: 문서의 주요 콘텐츠를 포함하며, 페이지당 한 번만 사용해야 한다.
  • <article>: 독립적으로 배포되거나 재사용될 수 있는 콘텐츠를 나타낸다 (예: 블로그 게시글).
  • <section>: 문서의 독립적인 구획을 나타낸다.
  • <aside>: 문서의 주요 콘텐츠와 간접적으로 관련된 콘텐츠를 포함한다. (예: 사이드바).
  • <footer>: 페이지의 꼬리말 정보를 담는다.

4.2. 텍스트 콘텐츠 태그

  • <h1> ~ <h6>: 제목을 나타내는 태그로, 숫자가 작을수록 중요한 제목이다.
  • <p>: 하나의 문단을 정의한다.
  • <a>: 다른 페이지로 이동하거나 특정 위치로 스크롤하는 하이퍼링크를 생성한다.
  • <strong>: 텍스트의 중요도를 강조한다. 시각적으로는 굵게 표시된다.
  • <em>: 텍스트의 강세를 강조한다. 시각적으로는 기울임꼴로 표시된다.
  • <div>: 특별한 의미 없는 일반적인 블록 요소로, 다른 요소를 묶어 스타일을 적용할 때 주로 사용한다.
  • <span>: 특별한 의미 없는 일반적인 인라인 요소로, 텍스트의 일부에 스타일을 적용할 때 주로 사용한다.

4.3. 미디어 및 폼 태그

  • <img>: 웹 페이지에 이미지를 삽입하는 태그이다. src 속성으로 이미지 경로를 지정한다.
  • <audio>: 오디오 콘텐츠를 삽입한다.
  • <video>: 비디오 콘텐츠를 삽입한다.
  • <form>: 사용자 입력을 위한 폼을 생성한다.
  • <input>: 사용자로부터 데이터를 입력받는 필드를 생성한다 (텍스트, 비밀번호, 체크박스 등).
  • <button>: 클릭 가능한 버튼을 생성한다.

5. HTML의 활용과 미래

HTML은 웹의 가장 근본적인 언어로서 다양한 분야에서 응용된다.

  • 정적 웹페이지 제작: 가장 기본적인 활용으로, 변경이 거의 없는 간단한 웹 페이지를 만들 때 사용된다.
  • 동적 웹 콘텐츠의 기반: React, Vue와 같은 SPA(Single Page Application) 프레임워크에서 HTML은 컴포넌트의 템플릿 역할을 수행한다.
  • 접근성과 웹 표준: 시맨틱 태그를 사용하여 문서의 의미를 명확히 함으로써, 검색 엔진 최적화(SEO)와 스크린 리더 같은 보조 기술에 도움을 준다.
